	查看分区：纯物理  fdisk -l /dev/sda

2-Linux系统的组成"内核”“内核模块”“系统调用”“用户空间”几者之间关系


3、Linux的文件结构与Windows的文件系统


4、/home普通用户所在目录，
	a、在该目录下以用户名为标志的文件夹区分，
	b、该目录若在独立分区时候占用空间要足够大，当前用户的所有信息都在该目录下

	c、该目录下的不同用户文件夹都具有自己的权限

	d、经常检查该目录下的用户的权限
       	   检查方式：入侵检测系统、find命令、命令历史记录

5、/root管理员用户所在的文件主目录，一个系统中管理员可以有多个，
     对安全要求高的系统必须使用多个管理员身份

	实际使用中这个目录一般会做更改！


6、/usr在没有/home之前普通用户所在的目录想在普通用户主目录均存放于/home下


7、/lib与/lib64系统根目录下的，关于C语言对应头文件C程序的二进制代码；---即公共函数库

	a、C语言使用的 #include中对应头文件的C代码的二进制  .so  .a文件等


	b、modules（模块）  系统的内核模块位于此处，在开发中进行相关内核模块开发时常用
	c、/boot为内核 ，/lib/modules下为内核模块



8、/usr/lib这里存放的是C语言编译的库文件.so  .a文件等



9、/usr/include C语言函数的函数原型的头文件在此处

	C语言中 #include<stdio.h>
	头文件：函数原型、类型定义、类型定义等

10、/usr/src

	默认为空，系统的源码在此（内核、驱动。。。）

11、/bin与/sbin

    /usr/bin与/usr/sbin

	在终端输入的命令如：ls、cd等命令
	sbin是在bin不够用的情况下开辟的

12、/temp
    /usr/temp

	文件开放权限最高，任何用户都可以在此操作


13、/dev系统设备的目录


14、/mnt外部设备的挂载目录


	dev是决定于系统的硬件设备以及对应驱动而存在的。如：硬盘、键盘等
	mnt是外界的其他设备：打印机、U盘等

15、/opt

	关于软件的四个组成部分：
		程序部分
		数据部分
		运行控制部分
		配置部分

	Unix下安装软件的过程中的程序部分位于此处
16、/var

	Unix下安装软件的数据部分存储在此
	运行是的日志（系统、软件）的都存在此目录下

17、/run

	Unix下安装软件的控制部分存储在此

	对于服务器类型软件进行进程间通信，软件间信息传递可以通过此处控制

18、/proc与/sys
		
    
	内核中的一部分数据存储在此，
	但是物理上是不存在的

	/sys可以理解为对/proc的文件按照分类进行规范

	查看一个文件信息：/proc/sys/fs/file-max 这个文件中的值表示的含义是关于
	系统中一个进程可以打开的最大描述符的数量！

19、/etc

	关于程序的第四部分配置部分，一般回存储在这里，但是不遵循此操作的也有很多

	a、/etc/passwd

		记录用户的基本信息
		以一行为单位，一行表示一个用户信息
		文件内容结构：以“：”进行分割

     root ：  x    :   0     :      0    :     root   :    /root       : /bin/bash

    用户名 保留之前  id号         组id     用户说明      用户工作主目录  解释器
           存储密码 0：管理员              可以不写    
           密文     非0：普通用户

	关于解释器的理解：某个用户登陆系统其执行的一些命令、脚本程序都通过解释器进行
			  执行

	b、/etc/shadow  文件开放权限比较高

		/etc/passwd文件的影子文件，记录的内容是密码加密后的密文

		cat /etc/shadow  查看密文内容

		密文的格式：  用户名：密码密文

	c、/etc/group

			用户组文件

		文件内容包括四部分，以冒号分隔

	root     :    x       :      0     :   xxxx

	组名        一般不用       组id       预留组名
	用户组组名  在网络环境中              多个用户名时，使用逗号隔开
	可重复      设置域的时候
		    需要使用

20、用户主目录的概念


	以当前用户为例

	.bash_history    当前用户输入的命令历史，防护安全
	.bash_logout     当用户退出时执行的操作可以写在这里，可以是脚本
	.bash_profile    用户登录进行时执行的东西
	.bashrc         用户登录时执行的脚本与早期的软件兼容

	两个登录时执行的文件，.bashrc优先于.bash_profile
